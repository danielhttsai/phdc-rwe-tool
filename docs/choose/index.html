Appraisal：評讀一個「藥物X 用 vs 不用」的研究

Database：小賴醫師要重做這個分析，需要對的資料

延續 Appraisal 的故事：小賴醫師想知道「某慢性病患者中，有用藥物X vs 沒用的人是不是活得比較久？」第一版（曾用藥＝暴露、從診斷起算）看起來「用藥的人死亡風險少 40%」，但那其實是不死時間灌出來的。要正確重做（只收新開始用藥的人、用藥那天才算暴露、大家的起算日對齊），資料裡就得有這些設計需要的東西。你手上是哪種資料庫？

Bias：這些研究各中了哪些偏誤？

同一個故事的延伸：先評讀小賴醫師最初那一版（第 1 題），再看幾個變形。每題勾出你認為存在的偏誤，按「對答案」看研究者哪裡跌倒、又該怎麼爬起來。

Method：小賴醫師的問題，該用哪種設計？

收尾：小賴醫師想正確回答「藥物X 用 vs 不用，誰活得久？」。這是一個「暴露隨時間開關、要避免不死時間」的問題 → 順著樹點下去，你會落在用藥那天才算暴露 / 主動對照新使用者 / 複製-中斷-加權 這一類「從開始用藥那天起算、大家對齊」的設計。試試看樹會不會帶你到同樣的答案。
