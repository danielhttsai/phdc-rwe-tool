Lilly RWE workshop 2026 / 07 / 22 一個小賴醫師都可能犯的錯：他發現「有吃藥物X 的人死亡風險少 40%」，但那 40% 是假的。四個分頁帶你把這個錯找出來、修好：Appraisal 讀出問題 · Database 找對資料 · Bias 認出偏誤 · Method 選對設計。

Appraisal：評讀一個「藥物X 用 vs 不用」的研究

Database：小賴醫師要重做這個分析，需要對的資料

小賴醫師知道第一版錯在哪了，想重做一次。這次他打算只收新開始用藥的人、用藥那天才算暴露、每個人從同一天起算。但這三件事都得資料裡真的有那些東西才做得到。你手上是哪一種資料庫？

Bias：這些研究各中了哪些偏誤？

換你當審查委員。先答六題認一下每種偏誤長什麼樣，再往下看幾個真實會遇到的情境，每題勾出你認為存在的偏誤，按「對答案」看研究者哪裡跌倒、又該怎麼爬起來。

Method：小賴醫師的問題，該用哪種設計？

最後一站：資料有了，那到底要用哪一種設計？先答六題，再往下用那棵樹走一次，看看你們會不會走到同一個答案。
